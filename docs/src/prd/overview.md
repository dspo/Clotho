# Clotho App - 产品需求文档（PRD）

## 1. 产品概述与定位

### 1.1 产品名称
Clotho（克洛托，希腊神话中命运三女神之一，掌管"纺织生命之线"）

### 1.2 产品定位
Clotho 是一款面向个人用户的轻量级桌面端项目管理/任务管理/日程管理工具。

**核心理念**：以个人视角出发，将项目、任务、日程三者有机整合，提供多种视图（列表、看板、甘特图、日历）满足不同场景下的管理需求。

### 1.3 目标用户
- 独立开发者、自由职业者
- 需要管理多个个人项目的知识工作者
- 希望用一个工具统一管理任务和日程的效率爱好者

### 1.4 技术栈
- **框架**：Tauri 2（Rust 后端 + Web 前端）
- **前端**：React + TypeScript
- **样式**：TailwindCSS + shadcn/ui
- **数据存储**：本地 SQLite（通过 Tauri 插件）

### 1.5 核心差异化
| 维度 | Todoist/TickTick | Notion | Linear/Plane | Clotho |
|------|-----------------|--------|-------------|--------|
| 定位 | 个人待办 | 全能工作空间 | 团队项目管理 | 个人项目管理 |
| 视图 | 列表+日历 | 数据库多视图 | 看板+列表+时间线 | 5种视图全覆盖 |
| 甘特图 | 无 | 时间线（弱） | 有（团队向） | 有（个人向） |
| 离线 | 部分 | 部分 | 否 | 完全离线 |
| 隐私 | 云端 | 云端 | 云端 | 纯本地存储 |
| 性能 | 一般 | 较重 | 快 | 原生级性能 |

## 2. 竞品分析总结

### 2.1 个人任务管理类

#### Todoist
- **核心功能**：任务 CRUD、项目/分区管理、优先级（P1-P4，颜色编码）、标签、截止日期（支持自然语言输入）、循环任务、子任务、评论、过滤器/视图
- **UI 布局**：左侧边栏（收藏夹、项目列表、标签、过滤器）+ 右侧主内容区
- **视图**：列表视图（默认）、看板视图
- **交互亮点**：快速添加（全局快捷键）、自然语言日期解析（"明天下午3点"）、拖拽排序、行内编辑、批量操作
- **任务字段**：标题、描述、优先级、标签、截止日期、项目、分区、子任务、评论、提醒
- **不足**：无甘特图、日历视图较弱、无时间段概念（只有截止日期）

#### TickTick
- **核心功能**：任务管理、日历集成、看板、习惯追踪、番茄钟、白噪音
- **UI 布局**：左侧边栏（智能列表：今天/明天/未来7天/全部 + 项目列表）+ 主内容区 + 可选右侧日历面板
- **视图**：列表、看板、日历（月/周/日/时间线）
- **交互亮点**：日历与任务深度集成（任务可直接拖到日历上安排时间）、多级子任务、任务时长设置、时间块规划
- **任务字段**：标题、描述、优先级（高/中/低/无）、标签、截止日期、提醒、重复、子任务、附件、时长
- **借鉴点**：日历与任务的融合方式、时间块规划

### 2.2 知识+任务管理类

#### Notion
- **核心功能**：数据库驱动的多视图系统、页面嵌套、模板、关联数据库
- **UI 布局**：左侧边栏（页面树 + 收藏夹）+ 主内容区（宽屏）
- **视图**：表格、看板、日历、时间线、列表、画廊
- **交互亮点**：数据库属性自定义、视图间无缝切换、筛选/排序/分组组合、关联与汇总、拖拽
- **任务字段**：完全自定义（通过数据库属性：文本、数字、选择、多选、日期、人员、关联等）
- **借鉴点**：多视图切换的 Tab 设计、属性面板、筛选器 UI

#### Obsidian
- **核心功能**：Markdown 笔记 + 插件生态（Kanban、Tasks、Dataview 等插件实现任务管理）
- **UI 布局**：左侧文件树 + 中间编辑区 + 右侧面板
- **借鉴点**：本地优先理念、快捷键驱动、命令面板（Cmd+P）

### 2.3 现代项目管理类

#### Linear
- **核心功能**：Issue 追踪、项目管理、Cycle（迭代）、Roadmap、Triage（分诊）
- **UI 布局**：左侧窄边栏（团队/项目导航）+ 主内容区 + Issue 详情侧面板
- **视图**：列表、看板、时间线
- **交互亮点**：
  - 极致的键盘操作体验（几乎所有操作都有快捷键）
  - Cmd+K 命令面板（全局搜索+命令执行）
  - 行内状态切换（点击状态图标直接切换）
  - 快速创建（C 键创建 Issue）
  - 批量选择和操作
  - 流畅的动画过渡
- **Issue 字段**：标题、描述（Markdown）、状态（Backlog/Todo/In Progress/Done/Cancelled）、优先级（Urgent/High/Medium/Low/No Priority）、标签、负责人、项目、Cycle、截止日期、估算点数、父 Issue、子 Issue
- **借鉴点**：键盘优先的交互设计、命令面板、状态流转模型、简洁的 UI 风格

#### Plane.so
- **核心功能**：Issue 追踪、Cycle（迭代）、Module（模块）、Page（文档）、多视图
- **UI 布局**：左侧边栏（工作空间/项目导航）+ 主内容区
- **视图**：看板、列表、电子表格、甘特图、日历
- **交互亮点**：开源可自托管、视图布局切换、筛选器组合、Issue 详情侧面板
- **Issue 字段**：标题、描述、状态（自定义状态组：Backlog/Unstarted/Started/Completed/Cancelled）、优先级（Urgent/High/Medium/Low/None）、标签、负责人、模块、Cycle、开始日期、截止日期、估算
- **借鉴点**：5种视图的完整实现、甘特图设计、开源架构参考

### 2.4 看板类

#### Trello
- **核心功能**：看板（Board）、列表（List）、卡片（Card）、Power-Up 插件
- **UI 布局**：顶部导航栏 + 全屏看板区域
- **视图**：看板（核心）、时间线、日历、表格、仪表盘（需付费）
- **交互亮点**：拖拽卡片和列表、快速编辑（点击卡片标题直接编辑）、标签颜色系统、清单（Checklist）进度条
- **卡片字段**：标题、描述、成员、标签（颜色+文字）、清单、截止日期、附件、封面图
- **借鉴点**：拖拽交互的流畅性、看板的经典布局

#### Focalboard
- **核心功能**：开源看板、多视图（Board/Table/Calendar/Gallery）、自定义属性
- **UI 布局**：左侧边栏（Board 列表）+ 主内容区 + 视图切换 Tab
- **视图**：看板、表格、日历、画廊
- **交互亮点**：属性自定义、视图切换、筛选/排序/分组
- **借鉴点**：开源实现参考、多视图切换模式

### 2.5 日历日程类

#### Fantastical
- **核心功能**：日历管理、任务管理、自然语言输入、多日历源聚合
- **UI 布局**：左侧迷你日历+日历列表 + 右侧主日历视图
- **视图**：日视图、周视图、月视图、年视图
- **交互亮点**：自然语言创建事件（"明天下午3点和小明开会"）、日历集合（Calendar Sets）、快速切换视图
- **借鉴点**：自然语言输入、日历视图的精美设计、日/周/月视图切换

#### Google Calendar
- **核心功能**：日程管理、多日历、提醒、目标
- **UI 布局**：左侧迷你日历+日历列表 + 右侧主日历视图
- **视图**：日、周、月、年、日程（Agenda）、4天
- **快捷键**：D（日）、W（周）、M（月）、Y（年）、A（日程）、T（今天）、J/K（前后导航）
- **交互亮点**：点击时间槽快速创建、拖拽调整时间和时长、侧面板快速预览
- **借鉴点**：视图切换快捷键、时间槽点击创建、拖拽调整

### 2.6 甘特图类

#### GanttProject
- **核心功能**：甘特图、任务依赖关系、里程碑、资源管理、关键路径
- **UI 布局**：左侧任务列表（树形）+ 右侧甘特图时间轴
- **交互亮点**：拖拽调整任务时间、依赖关系连线、缩放时间轴
- **任务字段**：名称、开始日期、结束日期、时长、完成百分比、依赖关系、里程碑、资源分配
- **借鉴点**：甘特图的经典布局（左表右图）、依赖关系可视化

### 2.7 竞品分析关键洞察

| 设计维度 | 行业共识 | Clotho 采纳 |
|---------|---------|------------|
| 布局模式 | 侧边栏 + 主内容区 | 采纳，左侧导航 + 右侧内容 |
| 视图切换 | Tab 栏或工具栏按钮 | 采纳，主内容区顶部 Tab 切换 |
| 任务创建 | 快速添加 + 详情面板 | 采纳，行内快速创建 + 侧面板详情 |
| 键盘操作 | Linear 级别的快捷键 | 采纳，全面快捷键 + Cmd+K 命令面板 |
| 状态模型 | 5 阶段状态流转 | 采纳，Backlog→Todo→In Progress→Done→Cancelled |
| 优先级 | 4 级优先级 | 采纳，Urgent/High/Medium/Low |
| 主题 | 亮色/暗色双主题 | 采纳 |
| 数据存储 | 云端为主 | 差异化：纯本地 SQLite |

## 3. 功能范围（MVP）

### 3.1 核心功能清单

#### P0 - 必须实现（MVP）

**项目管理**
- 创建项目（名称、描述、颜色/图标）
- 项目列表展示（侧边栏 + 项目列表视图）
- 项目归档/取消归档
- 项目删除（软删除，可恢复）
- 项目排序（手动拖拽、按名称、按创建时间）

**任务管理**
- 任务 CRUD（创建、读取、更新、删除）
- 任务状态流转：Backlog → Todo → In Progress → Done → Cancelled
- 任务优先级：Urgent（红）、High（橙）、Medium（黄）、Low（蓝）
- 标签系统（创建、编辑、删除、多标签关联）
- 截止日期设置
- 子任务（一级子任务）
- 任务描述（Markdown 富文本）
- 任务排序（手动拖拽、按优先级、按截止日期、按创建时间）

**视图系统**
- 项目列表视图：所有项目的概览，含进度统计
- 看板视图：按状态分列的卡片拖拽
- 列表视图：表格式任务列表，支持排序/筛选
- 甘特图视图：时间轴上的任务条，支持依赖关系
- 日历视图：月/周/日视图，任务按日期展示

**搜索与筛选**
- 全局搜索（Cmd+K 命令面板）
- 按状态、优先级、标签、截止日期筛选
- 筛选条件组合（AND 逻辑）

**快捷键**
- 全局快捷键（命令面板、快速创建、导航）
- 视图内快捷键（状态切换、选择、编辑）

**主题**
- 亮色/暗色主题切换
- 跟随系统主题

#### P1 - 重要但可延后

- 任务开始日期
- 任务时长估算
- 甘特图任务依赖关系（前置/后置）
- 日历视图拖拽调整日期
- 数据导入/导出（JSON）
- 任务模板
- 自定义筛选视图保存

#### P2 - 锦上添花

- 循环任务
- 番茄钟/专注计时
- 统计仪表盘（完成率、趋势图）
- 多语言支持
- 自定义任务字段

## 4. 数据模型设计

### 4.1 实体关系概览

```
Project 1──N Task
Task    1──N Task (子任务，parent_task_id)
Task    N──N Tag  (通过 TaskTag 关联表)
Task    1──N TaskDependency (甘特图依赖关系)
```

### 4.2 Project（项目）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 主键 |
| name | VARCHAR(255) | 是 | 项目名称 |
| description | TEXT | 否 | 项目描述 |
| color | VARCHAR(7) | 否 | 项目颜色（HEX，如 #3B82F6） |
| icon | VARCHAR(50) | 否 | 项目图标（emoji 或图标名） |
| status | ENUM | 是 | active / archived |
| sort_order | INTEGER | 是 | 排序序号 |
| created_at | DATETIME | 是 | 创建时间 |
| updated_at | DATETIME | 是 | 更新时间 |
| deleted_at | DATETIME | 否 | 软删除时间 |

### 4.3 Task（任务）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 主键 |
| project_id | UUID (FK) | 是 | 所属项目 |
| parent_task_id | UUID (FK) | 否 | 父任务 ID（子任务关系） |
| title | VARCHAR(500) | 是 | 任务标题 |
| description | TEXT | 否 | 任务描述（Markdown） |
| status | ENUM | 是 | backlog / todo / in_progress / done / cancelled |
| priority | ENUM | 是 | urgent / high / medium / low（默认 low） |
| start_date | DATE | 否 | 开始日期（甘特图用） |
| due_date | DATE | 否 | 截止日期 |
| completed_at | DATETIME | 否 | 完成时间 |
| sort_order | INTEGER | 是 | 排序序号（列表/看板内排序） |
| kanban_order | INTEGER | 是 | 看板内排序序号 |
| created_at | DATETIME | 是 | 创建时间 |
| updated_at | DATETIME | 是 | 更新时间 |
| deleted_at | DATETIME | 否 | 软删除时间 |

**状态流转规则**：
```
backlog ──→ todo ──→ in_progress ──→ done
  │           │          │              │
  └───────────┴──────────┴──→ cancelled │
                                        │
  任何状态 ←──────────────── 可重新打开 ←┘
```

**状态定义**：
- **Backlog**：待规划，尚未决定何时处理
- **Todo**：已规划，准备开始
- **In Progress**：进行中
- **Done**：已完成
- **Cancelled**：已取消（不再需要）

### 4.4 Tag（标签）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 主键 |
| name | VARCHAR(100) | 是 | 标签名称（唯一） |
| color | VARCHAR(7) | 是 | 标签颜色（HEX） |
| created_at | DATETIME | 是 | 创建时间 |

### 4.5 TaskTag（任务-标签关联）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| task_id | UUID (FK) | 是 | 任务 ID |
| tag_id | UUID (FK) | 是 | 标签 ID |

复合主键：(task_id, tag_id)

### 4.6 TaskDependency（任务依赖关系）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 主键 |
| predecessor_id | UUID (FK) | 是 | 前置任务 ID |
| successor_id | UUID (FK) | 是 | 后置任务 ID |
| dependency_type | ENUM | 是 | finish_to_start（默认）/ start_to_start / finish_to_finish / start_to_finish |
| created_at | DATETIME | 是 | 创建时间 |

唯一约束：(predecessor_id, successor_id)

### 4.7 AppSettings（应用设置）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| key | VARCHAR(100) | 是 | 设置键（主键） |
| value | TEXT | 是 | 设置值（JSON 序列化） |
| updated_at | DATETIME | 是 | 更新时间 |

常用设置项：
- `theme`：light / dark / system
- `sidebar_collapsed`：true / false
- `default_view`：kanban / list / gantt / calendar
- `language`：zh-CN / en-US

### 5.2 侧边栏设计

```
┌────────────┐
│  🔍 搜索    │ ← Cmd+K 触发命令面板
├────────────┤
│  ⭐ 收藏夹   │
│    项目 A   │
│    项目 B   │
├────────────┤
│  📁 项目     │ ← 可折叠
│    + 新项目  │
│    项目 A   │ ← 左侧颜色条标识
│    项目 B   │
│    项目 C   │
│    ···      │
│  📦 已归档   │ ← 折叠显示
├────────────┤
│  🏷 标签     │ ← 可折叠
│    Bug      │
│    Feature  │
│    ···      │
├────────────┤
│  ⚙ 设置     │
└────────────┘
```

**侧边栏行为**：
- 默认宽度：240px
- 可拖拽调整宽度（最小 200px，最大 400px）
- 支持折叠为图标模式（宽度 48px）

**任务右键菜单**：
- 打开详情

- 设置优先级 →（子菜单：4 种优先级）
- 添加标签 →
- 设置截止日期
- 复制任务
- 删除任务

- 打开项目
- 重命名
- 设置颜色 →
- 添加到收藏 / 从收藏移除
- 归档项目
- 删除项目

### 7.5 通知与反馈

**操作反馈**：
- 成功操作：底部 Toast 提示（自动消失，3秒）
- 删除操作：Toast 提示 + "撤销"按钮（5秒内可撤销）
- 错误操作：红色 Toast 提示 + 错误信息
- 加载状态：骨架屏（Skeleton）或 Spinner

**确认对话框**：
- 删除项目：需二次确认（"此操作将删除项目及其所有任务"）
- 删除任务：需确认（可勾选"不再提示"）
- 归档项目：轻量确认

### 7.6 响应式与性能

**性能目标**：
- 应用启动时间 < 1 秒
- 视图切换 < 200ms
- 任务创建/更新 < 100ms
- 拖拽操作 60fps 流畅
- 支持单项目 1000+ 任务不卡顿

**数据加载策略**：
- 列表/看板：分页加载或虚拟滚动（任务数 > 100 时）
- 甘特图：可视区域渲染 + 缓冲区
- 日历：按月加载数据
- 搜索：防抖 300ms + 本地 SQLite 全文索引

### 7.7 数据安全

- 所有数据存储在本地 SQLite 数据库
- 数据库文件位置：`~/.clotho/data.db`
- 支持手动导出/导入（JSON 格式）
- 软删除机制（deleted_at 字段），30 天后自动清理
- 操作历史记录（支持撤销/重做）
