# 甘特图视图 - 详细设计文档

## 1. 概述

甘特图视图是 Clotho 的核心视图之一，用于在时间轴上可视化任务的起止时间、持续时长、进度和依赖关系。采用经典的左右分栏布局：左侧为任务列表面板（树形结构），右侧为 SVG 渲染的时间轴甘特图。

技术实现基于自研 SVG 方案，时间计算使用 date-fns，大量任务时使用 TanStack Virtual 实现虚拟滚动。

### 2.2 布局规格

| 区域 | 宽度/高度 | 说明 |
|------|----------|------|
| 工具栏 | 高度 40px | 固定在顶部，包含导航和缩放控制 |
| 左侧任务列表面板 | 默认 320px，可调范围 200-500px | 树形任务列表 |
| 分割线 | 4px 宽 | cursor: col-resize，悬停时高亮（背景变为主题强调色 50% 不透明度） |
| 右侧甘特图面板 | flex: 1 填充剩余空间 | SVG 时间轴渲染 |
| 时间轴头部 | 高度 48px（双行各 24px） | sticky 固定在右侧面板顶部 |
| 任务行高 | 36px | 左右面板行高保持一致 |

### 2.3 左右面板同步滚动

- 左侧面板和右侧面板共享同一个**垂直滚动位置**
- 任何一侧的垂直滚动操作（鼠标滚轮、触摸板、滚动条拖拽）都会同步另一侧
- 右侧面板额外支持独立的水平滚动（时间轴方向）
- 实现方式：监听两侧的 `scroll` 事件，同步 `scrollTop` 值，使用 `requestAnimationFrame` 避免抖动

### 3.3 时间轴 SVG 结构

```tsx
<svg class="gantt-timeline" width={totalWidth} height={totalHeight}>
  {/* 背景层 */}
  <g class="gantt-bg">

  {/* 今日线 */}

  </g>

  {/* 任务条层 */}
  <g class="gantt-bars">
    {/* 各任务条和里程碑 */}
  </g>

  {/* 依赖连线层 */}
  <g class="gantt-dependencies">
    {/* 贝塞尔曲线连线 */}

```

### 3.4 今日线

- 样式：红色竖线（`#ef4444`），宽度 1.5px，虚线 `stroke-dasharray="4,2"`
- 顶部标记：红色倒三角形（SVG polygon），宽 10px，高 6px，紧贴时间轴头部下方
- 层级：位于任务条和依赖线之上（z 轴最高层）
- 始终贯穿整个甘特图高度

### 3.5 背景区分

| 区域类型 | 背景色（亮色主题） | 背景色（暗色主题） |
|---------|------------------|------------------|
| 普通工作日 | 透明（#ffffff） | 透明（#24243A） |
| 周末（周六、周日） | #f8f9fa | #1e1e2e |
| 当日列 | #eff6ff（浅蓝） | #1e293b（深蓝） |

### 3.6 缩放交互

- **Ctrl + 鼠标滚轮**（macOS 为 Cmd + 滚轮）：缩放时间轴级别
- 缩放以鼠标光标所在时间点为中心
- 缩放过渡动画 200ms，ease-out 缓动
- 工具栏 [+] [-] 按钮也可控制缩放
- 缩放级别标签实时显示当前级别（如「月」「周」等）

### 4.1 普通任务条 - SVG rect



### 4.2 任务条规格

| 属性 | 值 | 说明 |
|------|-----|------|
| 高度 | 24px | 固定高度 |
| 圆角 | 4px | rx, ry |
| 行间距 | 36px | 每行任务占 36px（上下各 6px 间距） |
| 颜色填充 | 项目颜色 | 取所属 Project 的 color 字段 |
| 背景不透明度 | 0.6 | 未完成部分 |
| 进度不透明度 | 1.0 | 已完成部分 |
| 最小宽度 | 8px | 即使时间跨度很短也保证可见可点击 |
| 悬停效果 | 亮度提升 + 阴影 | filter: brightness(1.1)，添加 drop-shadow |

### 4.3 拖拽手柄视觉设计

任务条左右边缘的拖拽手柄在悬停时显示视觉指示：

| 属性 | 值 | 说明 |
|------|-----|------|
| 手柄宽度 | 6px | 左右边缘各 6px 的可拖拽区域 |
| 视觉样式 | 2 条垂直短线 | 间距 2px，高度约 16px，居中对齐 |
| 默认状态 | 隐藏（opacity: 0） | 不显示手柄 |
| 悬停状态 | 半透明显示（opacity: 0.6） | 鼠标悬停在任务条上时显示 |
| 线条颜色 | 与任务条颜色协调 | 使用项目颜色（projectColor），保持视觉一致性 |
| 鼠标样式 | 左边缘 `w-resize`，右边缘 `e-resize` | 提示用户可拖拽方向 |

### 4.3 任务名称显示策略

- **条宽度 >= 120px**：名称显示在条内（进度文本右侧），颜色白色
- **条宽度 < 120px**：名称显示在条右侧外部，颜色为当前主题文字色
- 名称超出可用空间时截断，尾部显示省略号（`...`）

## 5. 拖拽交互

### 5.1 拖拽模式

| 拖拽位置 | 鼠标样式 | 操作 | 说明 |
|---------|---------|------|------|
| 条左边缘（6px 区域） | `w-resize` | 调整开始日期 | 结束日期不变，改变时长 |
| 条右边缘（6px 区域） | `e-resize` | 调整结束日期 | 开始日期不变，改变时长 |
| 条中间区域 | `grab` / `grabbing` | 整体移动 | 保持时长不变，改变起止日期 |

### 5.2 拖拽行为

- **吸附**：拖拽时自动吸附到日边界（最小粒度 1 天）
- **日期提示 tooltip**：拖拽过程中在鼠标附近显示当前日期值
  - 调整开始日期时：显示「开始：2026-01-08」
  - 调整结束日期时：显示「结束：2026-01-15」
  - 整体移动时：显示「01/08 ~ 01/15（8天）」
- **视觉反馈**：
  - 原始位置：保留半透明（0.3 不透明度）的原始条作为参考
  - 新位置：实色显示（正常不透明度）的预览条
- **约束**：
  - 开始日期不能晚于结束日期
  - 任务条最小时长 1 天
  - 如果存在依赖关系，拖拽时检查是否违反依赖约束（前置任务结束 <= 后置任务开始）

### 5.4 拖拽实现要点

由于甘特图区域是 SVG 渲染，拖拽交互使用**原生 SVG 拖拽**（pointerdown / pointermove / pointerup 事件），而非 dnd-kit：

1. `pointerdown`：记录起始位置、拖拽模式（左边缘/右边缘/整体）
2. `pointermove`：计算偏移量，吸附到日边界，更新预览条位置，显示 tooltip
3. `pointerup`：提交日期变更，触发数据更新
4. 使用 `setPointerCapture` 确保拖拽过程中鼠标移出元素仍能追踪

## 6. 依赖关系

### 6.1 连线样式

| 属性 | 值 | 说明 |
|------|-----|------|
| 线型 | 贝塞尔曲线（SVG path，cubic bezier） | 平滑弯曲 |
| 颜色 | #94a3b8 | 默认灰色 |
| 宽度 | 1.5px | stroke-width |
| 箭头 | 终点处三角箭头 | SVG marker-end |
| 悬停颜色 | #3b82f6（蓝色） | 高亮效果 |
| 悬停宽度 | 2.5px | 加粗 |

### 6.2 依赖类型

默认使用 FS（Finish-to-Start）类型：前置任务完成后，后置任务才能开始。

| 类型 | 缩写 | 含义 | 连线方向 |
|------|------|------|---------|
| 完成-开始 | FS | 前置完成 → 后置开始 | 前置右端 → 后置左端 |
| 开始-开始 | SS | 前置开始 → 后置开始 | 前置左端 → 后置左端 |
| 完成-完成 | FF | 前置完成 → 后置完成 | 前置右端 → 后置右端 |
| 开始-完成 | SF | 前置开始 → 后置完成 | 前置左端 → 后置右端 |

MVP 阶段仅实现 FS 类型，其余类型作为 P1 功能。

### 6.3 连线 SVG 结构

```tsx
{/* 箭头 marker 定义 */}
<defs>
  <marker id="arrow" markerWidth={8} markerHeight={6}
          refX={8} refY={3} orient="auto">
    <polygon points="0,0 8,3 0,6" fill="#94a3b8" />
  </marker>
  <marker id="arrow-hover" markerWidth={8} markerHeight={6}
          refX={8} refY={3} orient="auto">
    <polygon points="0,0 8,3 0,6" fill="#3b82f6" />
  </marker>
</defs>

{/* FS 依赖连线 */}
<path
  d={`M ${fromX},${fromY} C ${fromX + dx},${fromY} ${toX - dx},${toY} ${toX},${toY}`}
  fill="none"
  stroke="#94a3b8"
  stroke-width={1.5}
  marker-end="url(#arrow)"
/>
```

**贝塞尔曲线计算（FS 类型）**：
- 起点（fromX, fromY）：前置任务条右端中点
- 终点（toX, toY）：后置任务条左端中点
- 控制点偏移 dx = min(|toX - fromX| / 2, 40px)
- 如果 toX < fromX（后置任务在前置任务左边），连线需要绕行：先向右延伸 → 向下弯曲 → 向左到达终点

### 6.4 创建依赖关系

1. **悬停触发**：鼠标悬停在任务条末尾时，显示**连接点**（小圆圈，半径 5px，蓝色边框）
2. **开始拖拽**：从连接点拖出，显示一条跟随鼠标的临时连线（蓝色虚线）
3. **目标选择**：拖拽到另一个任务条上时，目标条高亮显示蓝色边框
4. **完成连接**：释放鼠标，创建依赖关系，显示正式连线
5. **取消**：释放在非任务条区域，取消操作

```
悬停任务条末尾:
     ┌──────────────────────┐○    ← 连接点（小圆圈）
     │▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░░░│
     └──────────────────────┘

拖拽创建依赖:
     ┌──────────────────────┐○- - - - - - - ╮
     │▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░░░│               │  ← 蓝色虚线
     └──────────────────────┘               ▼
                              ┌════════════════════════┐  ← 目标条蓝色边框高亮
                              │▓▓▓▓▓▓▓▓▓▓▓░░░░░░░░░░░░│
                              └════════════════════════┘
```

### 6.5 删除依赖关系

- 悬停连线时：连线高亮变蓝，连线中点附近显示删除按钮（红色 × 图标，16x16px）
- 点击删除按钮：弹出确认提示后删除依赖关系
- 快捷操作：选中连线后按 Delete 键也可删除

### 6.6 循环依赖检测

- 创建依赖时实时检测是否形成循环依赖（A → B → C → A）
- 如果检测到循环：
  - 临时连线变为红色虚线
  - 显示 tooltip 提示「无法创建：会形成循环依赖」
  - 释放鼠标时不创建依赖
- 检测算法：从目标任务出发进行 DFS，检查是否能回到起始任务

### 7.2 左侧项目列表行高规则

左侧任务列表按项目分组展示，每个项目区域的行高需与右侧甘特图对齐：

- **最小行数**：每个项目区域最少显示 3 行
- **并行任务 > 3 时**：行数 = 该项目的并行任务数量（即同一时间段内重叠的任务数）
- **高度对齐**：左侧项目区域高度 = 右侧对应项目任务区域的高度，确保左右面板严格对齐
- **无折叠功能**：左侧项目列表不需要折叠/展开功能，始终展开显示所有任务

## 8. 关键路径

### 8.1 开关控制

- 工具栏中提供「关键路径」开关按钮（Toggle 样式）
- 默认关闭
- 开启/关闭时平滑过渡动画 300ms

### 8.2 视觉效果

| 元素 | 关键路径开启时 | 非关键路径 |
|------|-------------|-----------|
| 任务条 | 加粗红色边框（2px，`#ef4444`） | 不透明度降至 0.4 |
| 依赖连线 | 变为红色（`#ef4444`），宽度 2px | 不透明度降至 0.3 |
| 里程碑 | 红色边框 | 不透明度降至 0.4 |

### 8.3 关键路径计算

- 算法：基于 CPM（Critical Path Method）
- 计算逻辑建议放在 Rust 侧（通过 Tauri command 调用），避免前端大量计算
- 输入：所有任务的开始日期、结束日期、依赖关系
- 输出：关键路径上的任务 ID 列表
- 触发时机：开启关键路径开关时计算一次，任务日期或依赖关系变更时重新计算

## 11. 无数据与边界状态

### 11.1 无任务

当项目无任务或无任何任务设置了日期时：

```
┌──────────────────────────────────────────────────────┐
│                                                      │
│              📅                                      │
│              暂无可显示的任务                           │
│              为任务设置开始日期和截止日期后               │
│              即可在甘特图中查看                         │
│                                                      │
│              [+ 创建任务]                              │
│                                                      │
└──────────────────────────────────────────────────────┘
```

### 11.2 仅有截止日期无开始日期

- 只有 `due_date` 没有 `start_date` 的任务：以截止日期为结束日期，开始日期默认为截止日期前 1 天
- 显示为最小宽度的任务条，条内不显示进度
- 左侧列表中开始日期列显示为灰色斜体「(推算)」

### 11.3 任务数量过多

- 超过 100 个可见任务行时，启用 TanStack Virtual 虚拟滚动
- 仅渲染可视区域 + 上下各 5 行缓冲区的任务条
- SVG 元素动态添加/移除，保持 DOM 节点数量可控
