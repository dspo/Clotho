# 日历日程视图 - 详细设计文档

## 1. 概述

日历视图是 Clotho 的五大核心视图之一，提供基于时间维度的任务与日程管理。用户可在月/周/日三种子视图间切换，直观地查看任务截止日期和日程事件在时间轴上的分布。

技术实现基于 Schedule-X 日历库（MIT 开源），通过其 React adapter 集成，利用 CSS 变量与 TailwindCSS/shadcn 主题系统对接。拖拽交互使用 Schedule-X 内置的 drag-and-drop 插件。

---

## 2. 入口与导航

### 2.1 入口方式

- **项目内入口**：项目详情页顶部视图切换 Tab 中的「日历」Tab（快捷键 Cmd+4）
- **全局入口**：侧边栏底部可放置全局日历入口，聚合所有项目的任务和日程

### 2.2 URL 路由

```
/projects/:projectId/calendar           # 项目内日历（默认月视图）
/projects/:projectId/calendar?view=week  # 项目内周视图
/projects/:projectId/calendar?view=day&date=2026-03-02  # 项目内日视图，指定日期
```

搜索参数：
- `view`：month（默认）| week | day
- `date`：当前查看的日期锚点，ISO 格式（YYYY-MM-DD）

| 元素 | 位置 | 行为 |
|------|------|------|
| ◀ 前导航 | 左侧 | 月视图：上月；周视图：上周；日视图：前一天 |
| ▶ 后导航 | 左侧，◀ 右侧 | 月视图：下月；周视图：下周；日视图：后一天 |
| 今天按钮 | 导航按钮右侧 | 跳转到包含今天的视图 |
| 日期范围标题 | 中部偏左 | 月视图："2026年3月"；周视图："2026年3月2日 - 3月8日"；日视图："2026年3月2日 星期一" |
| 视图切换按钮组 | 右侧 | 三个按钮：日 / 周 / 月，当前视图高亮，使用 shadcn ToggleGroup 组件 |

### 3.3 快捷键

| 快捷键 | 功能 |
|--------|------|
| M | 切换到月视图 |
| W | 切换到周视图 |
| D | 切换到日视图 |
| T | 跳转到今天 |
| ← / H | 前导航 |
| → / L | 后导航 |

### 4.2 网格规格

| 属性 | 值 |
|------|-----|
| 行数 | 6 行（固定，不足补前月/后月日期） |
| 列数 | 7 列（周一至周日） |
| 每格最小高度 | 100px |
| 每格内边距 | 4px |
| 日期数字位置 | 左上角，font-size: 13px |
| 网格线 | 1px solid，亮色模式 border-gray-200，暗色模式 border-gray-700 |

### 4.3 日期格子内容

**日期数字**：
- 普通日期：灰色文字（text-muted-foreground）
- 今天：数字外包裹 24px 圆形背景，背景色 primary（#3B82F6 亮色 / #60A5FA 暗色），文字白色
- 非当月日期：数字及所有内容 opacity: 0.4

**事件条**：
- 最多显示 3 个事件条（含跨天事件占位）
- 每条高度 20px，圆角 4px，水平间距撑满格子宽度（左右各留 2px）
- 垂直间距 2px
- 背景色：所属项目颜色（如 #3B82F6），文字白色，font-size: 12px
- 文字超出时 text-overflow: ellipsis
- 已完成任务：opacity: 0.5，标题加删除线

**跨天事件**：
- 横跨多个格子的连续色条
- 起始格子：左侧圆角，显示事件标题
- 中间格子：无圆角，无文字
- 结束格子：右侧圆角，无文字
- 跨周时在下一行继续显示（左侧平头）

**任务截止日期标记**：
- 在日期数字右侧显示 6px 圆形标记点
- 颜色为项目颜色
- 多个任务截止时最多显示 3 个点，水平排列

**"+N more" 链接**：
- 当事件数 > 3 时，第 3 个位置替换为 "+N more" 文字链接
- 文字颜色 text-muted-foreground，font-size: 11px
- 点击展开当天所有事件的浮层列表

### 4.4 月视图交互

| 交互 | 行为 |
|------|------|
| 点击日期数字 | 切换到该日的日视图 |
| 点击事件条 | 打开右侧任务详情面板 |
| 点击 "+N more" | 弹出当天事件列表浮层 |
| 双击空白区域 | 打开事件创建弹窗，日期预填为该格日期 |
| 拖拽事件条到其他日期格 | 修改事件日期（截止日期或事件日期） |
| 右键事件条 | 打开任务右键菜单 |
| 鼠标悬停事件条 | 显示 tooltip：标题、时间、项目名、状态 |

### 5.2 全天事件区域

| 属性 | 值 |
|------|-----|
| 位置 | 列头下方，时间网格上方 |

- 标记为"全天"的日程事件

- 仅有截止日期的任务（无具体时间段）在全天区域不显示，仅在月视图以标记点体现

### 5.3 时间网格

| 属性 | 值 |
|------|-----|
| 左侧时间刻度 | 00:00 - 23:00，每小时一行标签 |
| 时间刻度宽度 | 60px |

| 可视工作时间 | 8:00 - 18:00 高亮显示（非工作时间区域微弱灰色背景） |

### 5.4 时间段事件块

| 属性 | 值 |
|------|-----|
| 位置计算 | top = (startHour * 60 + startMinute) / 60 * 格高度 |
| 高度计算 | height = durationMinutes / 60 * 格高度，最小高度 20px（对应 20 分钟） |
| 宽度 | 列宽 - 8px（左右各留 4px 间距） |
| 圆角 | 6px |
| 背景色 | 项目颜色，opacity: 0.9 |
| 左侧边框 | 4px solid 项目颜色（深一个色阶），作为视觉强调 |
| 文字颜色 | 白色（深色背景时）或深色（浅色背景时），自动计算对比度 |
| 内容显示 | 标题（font-size: 12px, font-weight: 500）+ 时间范围（font-size: 11px, text-muted） |

**重叠事件处理**：
- 时间段有重叠的事件并排显示
- 2 个重叠：各占列宽 50%
- 3 个重叠：各占列宽 33%
- 4+ 个重叠：各占列宽 25%，超出部分水平滚动或以 "+N" 提示
- 并排时保留 2px 间距

**已完成任务显示**：
- opacity: 0.4
- 标题添加删除线（text-decoration: line-through）
- 背景色降低饱和度

## 7. 事件/任务显示规则

### 7.1 数据来源

日历上显示两类数据：

**任务（Task）**：
- 来自 Task 数据表
- 有 `due_date` 字段的任务在日历上显示
- 有 `start_date` + `due_date` 的任务可显示为时间段

**日程事件（Calendar Event）**：
- 未来可扩展独立的 CalendarEvent 数据表
- MVP 阶段，日历上仅显示任务，暂不支持独立的日程事件创建
- 任务的 `due_date` 作为日历上的时间锚点

### 7.2 显示映射规则

| 任务数据条件 | 月视图显示 | 周/日视图显示 |
|-------------|-----------|-------------|
| 仅有 due_date（无时间） | 日期格内的事件条 | 全天事件区域的事件条 |
| 有 due_date + 具体时间 | 日期格内的事件条 | 时间网格中对应位置的事件块（默认 1 小时时长） |
| 有 start_date + due_date（跨天） | 跨格子的横向长条 | 全天事件区域的事件条 |
| 有 start_date + due_date（同天） | 日期格内的事件条 | 时间网格中的时间段色块 |
| 状态为 done | 事件条半透明 + 删除线标题 | 事件块半透明 + 删除线标题 |
| 状态为 cancelled | 不显示（可通过筛选器开启） | 不显示 |
| 无 due_date 且无 start_date | 不显示 | 不显示 |

### 7.3 事件条颜色

- 背景色 = 所属项目的 `color` 字段值
- 如项目无颜色，使用默认色 primary（#3B82F6）
- 事件条左侧 4px 加深色边框作为项目标识

### 7.4 优先级标记

- 在事件条/块的标题前显示优先级小图标（圆形，4px）
- Urgent: 红色圆点
- High: 橙色圆点
- Medium: 黄色圆点
- Low: 不显示标记（减少视觉噪音）

## 8. 拖拽交互

### 8.1 拖拽能力总览

| 视图 | 拖拽操作 | 效果 | 数据变更 |
|------|---------|------|---------|
| 月视图 | 事件条 → 其他日期格 | 修改日期 | 更新 due_date |
| 周视图 | 事件块 → 上下移动 | 修改时间 | 更新时间字段 |
| 周视图 | 事件块 → 其他列 | 修改日期 | 更新 due_date |
| 周视图 | 事件块底部边缘 | 调整时长 | 更新结束时间 |
| 日视图 | 事件块 → 上下移动 | 修改时间 | 更新时间字段 |
| 日视图 | 事件块底部边缘 | 调整时长 | 更新结束时间 |
| 周/日 | 拖拽空白区域 | 选择时间范围 | 触发创建弹窗 |

### 8.2 拖拽视觉反馈

| 状态 | 视觉表现 |
|------|---------|
| 拖拽开始 | 原位置保留半透明占位（opacity: 0.3） |
| 拖拽中 | 跟随鼠标的半透明事件块副本（opacity: 0.7），显示预计放置的时间 |
| 有效放置区域 | 目标格子/时间段浅蓝色高亮 |
| 无效放置区域 | 鼠标变为禁止图标 |
| 释放后 | 事件平滑动画到新位置（transition: 200ms ease） |

### 8.3 时间吸附

- 周/日视图拖拽时，时间吸附到 15 分钟刻度
- 例：拖拽到 9:07 位置 → 自动吸附到 9:00；拖拽到 9:08 → 吸附到 9:15
- 调整时长时同样吸附到 15 分钟

## 11. 与其他视图联动

### 11.1 数据同步

日历视图与看板、列表、甘特图共享同一数据源（Zustand Store + SQLite）。任何视图中的数据变更实时反映到日历视图。

| 操作来源 | 日历视图响应 |
|---------|-------------|
| 看板中修改任务截止日期 | 日历上事件位置更新 |
| 列表中修改任务状态为 Done | 日历上事件样式变为半透明 + 删除线 |
| 甘特图中拖拽修改 start_date/due_date | 日历上事件日期范围更新 |
| 日历中拖拽修改日期 | 看板/列表/甘特图中对应字段更新 |

### 11.2 详情面板联动

- 点击日历上的任务事件 → 打开右侧任务详情侧面板（与其他视图共用同一面板组件）
- 详情面板中修改任何字段 → 日历实时更新
- 详情面板宽度 480px，从右侧滑入，不覆盖日历内容（日历区域自动收窄）

### 11.3 筛选器联动

日历视图复用全局筛选器组件：

- 按项目筛选：仅显示选中项目的任务
- 按优先级筛选：仅显示选中优先级的任务
- 按标签筛选：仅显示带有选中标签的任务
- 按状态筛选：默认隐藏 Cancelled 状态任务，可手动开启

## 12. Schedule-X 集成方案

### 12.1 使用的 Schedule-X 包

```
@schedule-x/react           # React adapter
@schedule-x/theme-default   # 默认主题（基础样式）
@schedule-x/calendar        # 核心日历引擎
@schedule-x/drag-and-drop   # 拖拽插件
@schedule-x/event-modal     # 事件弹窗插件（可选，可能自研替代）
@schedule-x/resize          # 事件调整大小插件
@schedule-x/current-time    # 当前时间线插件
```

### 12.2 主题定制

通过 CSS 变量覆盖 Schedule-X 默认主题，与 shadcn/ui 主题系统对齐：

```css
/* 亮色模式 */
:root {
  --sx-color-primary: var(--primary);           /* 主色调 */
  --sx-color-on-primary: var(--primary-foreground);
  --sx-color-surface: var(--background);        /* 背景 */
  --sx-color-on-surface: var(--foreground);     /* 文字 */
  --sx-color-surface-variant: var(--muted);     /* 次级背景 */
  --sx-color-on-surface-variant: var(--muted-foreground);
  --sx-internal-border: var(--border);          /* 边框 */
}

/* 暗色模式通过 .dark 类自动切换 */
```

### 12.3 自定义事件渲染

使用 Schedule-X 的 custom event renderer（自定义事件组件），替换默认事件样式：

- 事件条/块使用自定义 React 组件渲染
- 颜色由项目颜色决定（非 Schedule-X 默认颜色）
- 支持显示优先级图标、项目标识、已完成状态

### 12.4 事件弹窗替代

不使用 Schedule-X 内置的 event-modal 插件，自研弹窗组件以：
- 与 shadcn/ui Dialog 风格统一
- 支持更多自定义字段（项目选择、优先级等）
- 复用 Clotho 的表单组件

## 13. 状态管理

### 13.1 Calendar Store (Zustand)

```typescript
interface CalendarStore {
  // 视图状态
  currentView: 'month' | 'week' | 'day'
  currentDate: Date           // 当前查看的日期锚点

  // 操作
  setView: (view: 'month' | 'week' | 'day') => void
  setDate: (date: Date) => void
  navigateForward: () => void  // 根据当前视图前进（月/周/日）
  navigateBackward: () => void
  goToToday: () => void
}
```

### 13.2 数据获取

- 日历视图按可视日期范围从 SQLite 查询任务
- 月视图：查询当月 +/- 1 周的任务（覆盖前后月溢出的日期）
- 周视图：查询当周的任务
- 日视图：查询当天的任务
- 视图切换或日期导航时重新查询

## 14. 响应式与性能

### 14.1 性能目标

| 指标 | 目标值 |
|------|--------|
| 视图切换（月/周/日） | < 150ms |
| 日期导航（前/后） | < 100ms |
| 拖拽操作帧率 | 60fps |
| 单月事件加载 | < 200ms（1000 条以内） |

### 14.2 优化策略

- 按可视范围查询数据，避免全量加载
- 事件渲染使用 React.memo 避免不必要的重渲染
- 拖拽过程中使用 CSS transform 而非 DOM 重排
- 月视图中 "+N more" 浮层使用懒加载
- 迷你日历的事件标记点数据与主视图共享，不额外查询

### 14.3 窗口尺寸适配

| 窗口宽度 | 适配策略 |
|---------|---------|
| >= 1200px | 完整显示，侧边栏 + 日历 + 详情面板可并排 |
| 900-1200px | 详情面板覆盖日历区域（overlay 模式） |
| < 900px | 侧边栏自动折叠，日历全宽显示 |
