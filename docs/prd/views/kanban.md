# 看板视图 - 详细设计文档

## 1. 概述

看板视图是 Clotho 的核心视图之一，采用经典的列式布局，默认按任务状态分列展示任务卡片。用户可通过拖拽卡片在列间移动以改变任务状态，或在列内拖拽调整排序。

技术实现基于 dnd-kit（@dnd-kit/core + @dnd-kit/sortable），碰撞检测使用 closestCorners 策略。

## 3. 列（Column）设计

### 3.1 列定义

**默认分组：按任务状态**

看板默认按 5 种任务状态分列，列顺序固定：

| 列序号 | 状态 | 列名 | 状态颜色 |
|-------|------|------|---------|
| 1 | backlog | Backlog | #6B7280（灰） |
| 2 | todo | Todo | #8B5CF6（紫） |
| 3 | in_progress | In Progress | #F59E0B（琥珀） |
| 4 | done | Done | #10B981（绿） |
| 5 | cancelled | Cancelled | #EF4444（红） |

**自定义分组**

用户可通过顶部工具栏的「分组」下拉菜单切换分组维度：

- **按状态**（默认）：5 列，顺序固定
- **按优先级**：分为 Urgent / High / Medium / Low 四列
- **按标签**：每个标签一列 + 一个「无标签」列。标签列的顺序按标签创建时间排列，「无标签」列固定在最后

切换分组维度时，卡片重新归类到对应的列中，排序保持 `kanban_order` 字段值。

### 3.6 ProjectGroup 拖拽（按状态分组时）

当 `groupBy='status'` 时，每个状态列内的任务按所属项目（Project）分组显示为 ProjectGroup：

**布局**：
- 每个状态列内，任务按 Project 分组，显示项目名称作为组标题
- 组标题样式：项目颜色圆点 + 项目名称，字号 12px，字重 600

**拖拽规则**：
- ProjectGroup 内的任务支持拖拽排序（列内排序）
- **跨状态列拖拽**：将任务从一个状态列拖到另一个状态列，自动更新任务的 `status` 字段
- **跨 ProjectGroup 拖拽**：将任务从一个项目分组拖到另一个项目分组，自动更新任务的 `project_id` 字段
- 同时跨列和跨组时，同时更新 `status` 和 `project_id`

**技术实现**：
- 每个 ProjectGroup 内的任务列表需要独立的 `SortableContext` 包装
- 结构示意：
```
DndContext
├── SortableContext (列容器 - 水平)
│   ├── Column (Backlog)
│   │   ├── ProjectGroup (项目 A)
│   │   │   └── SortableContext (卡片 - 垂直)
│   │   │       ├── TaskCard
│   │   │       └── TaskCard
│   │   ├── ProjectGroup (项目 B)
│   │   │   └── SortableContext (卡片 - 垂直)
│   │   │       └── TaskCard
│   │   └── ...
│   ├── Column (Todo)
│   │   └── ...
│   └── ...
└── DragOverlay
```

**列头规格**：

| 属性 | 值 |
|------|-----|
| 列头高度 | 40px |
| 列头内边距 | 8px 12px（上下 左右） |
| 列头背景 | 与列背景相同（无额外背景色） |
| 列名字号 | 14px |

| 状态圆点 | 8px 直径，填充对应状态颜色 |

| 菜单按钮 | 16px 图标（三点），hover 时显示背景 |

**列头交互**：
- 状态圆点：纯展示，不可交互
- 列名：双击进入行内编辑模式（仅自定义分组模式下可编辑）。按状态分组时列名不可编辑
- 计数：实时更新，展示当前列中可见卡片数量
- 折叠按钮：点击折叠/展开列，快捷键无（通过菜单或点击）
- 菜单按钮（`...`）：点击弹出列操作菜单

**列操作菜单内容**：
- 折叠列
- 设置 WIP 限制... （弹出输入框，输入数字，0 表示无限制）
- 全选本列任务
- ---（分隔线）
- 新建列（仅自定义分组模式）
- 删除列（仅自定义分组模式，需确认，列中任务移到「无分组」列）

### 3.3 WIP（Work In Progress）限制

WIP 限制允许用户为每列设置最大任务数量，超出时视觉警告。

**设置方式**：列头菜单 > 设置 WIP 限制，输入数字（0 = 无限制，默认无限制）

**超出 WIP 时的视觉变化**：
- 列头计数数字变为红色（#EF4444）
- 列头计数背景变为浅红色（亮：#FEE2E2，暗：#7F1D1D）
- 列顶部出现 2px 红色边框线
- 计数格式变为 `5/3`（当前数/限制数）

```
正常状态：          超出 WIP：
● In Progress  2   ● In Progress  5/3
                   ──────────────────── ← 红色顶部线
```

### 3.5 列的拖拽排序

仅在自定义分组模式下，列支持拖拽排序。按状态分组时列顺序固定，不可拖拽。

**拖拽触发**：按住列头区域拖拽
**拖拽反馈**：被拖拽的列整体半透明（opacity: 0.5），目标位置显示 3px 宽的蓝色竖线（#3B82F6）
**释放后**：列平滑移动到目标位置（200ms ease 动画）

### 4.3 优先级色条

卡片左侧有一个 3px 宽的竖向色条，高度撑满卡片，用于表示任务优先级。

| 优先级 | 颜色 |
|--------|------|
| Urgent | #EF4444（红） |
| High | #F97316（橙） |
| Medium | #EAB308（黄） |
| Low | #3B82F6（蓝） |

色条通过 `border-left: 3px solid <color>` 实现，圆角跟随卡片左侧圆角。

### 4.4 卡片内容区域

卡片内容从上到下依次排列：

**第一行：任务标题**
- 字号：14px
- 字重：500 (medium)
- 颜色：主文字色
- 最多显示 2 行，超出截断显示省略号（`-webkit-line-clamp: 2`）
- 行高：1.4

**第二行：标签 + 截止日期**（有标签或截止日期时才显示此行）
- 布局：flex，标签左对齐，截止日期右对齐（`justify-content: space-between`）
- 标签 chips：
  - 字号：11px
  - 内边距：2px 6px
  - 圆角：4px
  - 背景色：标签颜色的 10% 透明度
  - 文字色：标签颜色
  - 最多显示 2 个标签，超出显示 `+N`
  - 标签之间间距 4px
- 截止日期：
  - 字号：12px
  - 颜色：次要文字色
  - 图标：日历图标（Lucide `Calendar`，12px）
  - 过期任务：文字和图标变为红色（#EF4444）
  - 今天截止：文字变为琥珀色（#F59E0B）
  - 格式：`MM/DD`（如 `12/25`），跨年显示 `YYYY/MM/DD`

**第三行：子任务进度条**（有子任务时才显示此行）
- 进度条高度：4px
- 进度条圆角：2px
- 进度条背景：亮：#E5E7EB，暗：#4B5563
- 进度条填充色：#10B981（绿）
- 进度文字：`已完成/总数`（如 `3/5`），字号 11px，次要文字色
- 进度条和文字在同一行，进度条占满剩余宽度，文字在右侧

**各行间距**：8px

### 4.5 卡片状态变体

**Done 状态的卡片**：
- 标题添加删除线（`text-decoration: line-through`）
- 整体透明度降低为 0.7
- 优先级色条变为灰色（#D1D5DB）

**Cancelled 状态的卡片**：
- 标题添加删除线
- 整体透明度降低为 0.5
- 优先级色条变为灰色（#D1D5DB）

### 4.6 卡片悬停效果

| 状态 | 效果 |
|------|------|
| 默认 | shadow: 0 1px 2px rgba(0,0,0,0.05) |

## 5. 拖拽交互（dnd-kit）

### 5.1 拖拽类型

看板中有三种拖拽场景：

| 场景 | 拖拽对象 | 数据变更 |
|------|---------|---------|
| 列间拖拽 | 卡片从 A 列拖到 B 列 | 更新任务状态 + kanban_order |
| 列内排序 | 卡片在同一列内上下移动 | 更新 kanban_order |
| 列排序 | 列头拖拽改变列位置 | 更新列顺序（仅自定义分组） |

### 5.2 dnd-kit 配置

```
DndContext
├── sensors: [PointerSensor, TouchSensor, KeyboardSensor]
├── collisionDetection: closestCorners
├── onDragStart → 设置拖拽状态，显示 DragOverlay
├── onDragOver → 实时更新卡片位置预览
├── onDragEnd → 提交数据变更
│
├── SortableContext (列容器 - 水平)
│   ├── Column (Backlog)
│   │   └── SortableContext (卡片 - 垂直)
│   │       ├── TaskCard
│   │       ├── TaskCard
│   │       └── ...
│   ├── Column (Todo)
│   │   └── SortableContext (卡片 - 垂直)
│   └── ...
│
└── DragOverlay → 拖拽中的卡片副本
```

**Sensor 配置**：
- PointerSensor：`activationConstraint: { distance: 5 }`（移动 5px 后才触发拖拽，避免误触）
- TouchSensor：`activationConstraint: { delay: 200, tolerance: 5 }`（长按 200ms 触发，见 5.6）
- KeyboardSensor：支持 Tab 聚焦 + Space 激活 + 方向键移动

当拖拽卡片接近列的顶部或底部边缘时（距离 < 60px），列内容自动滚动：
- 滚动速度：与距边缘的距离成反比，最大 600px/s
- 滚动方向：接近顶部向上滚动，接近底部向下滚动
- 过渡：平滑滚动（`scroll-behavior: smooth` 不适用，使用 requestAnimationFrame）

当拖拽卡片接近看板区域的左右边缘时（距离 < 80px），看板水平滚动：
- 规则同上，方向为左右
- 优先级：垂直滚动和水平滚动可以同时发生

### 5.6 触摸设备支持

| 手势 | 操作 |
|------|------|
| 长按 200ms | 触发拖拽模式（卡片轻微放大 + 震动反馈 `navigator.vibrate(50)`） |
| 拖拽中移动 | 卡片跟随手指 |
| 松开手指 | 释放到目标位置 |
| 点按（非长按） | 正常点击行为（打开详情） |

长按激活后的视觉反馈：卡片背景色变为选中色，200ms 后进入拖拽模式。

### 5.7 拖拽数据变更

**列间拖拽**：
1. 更新任务的 `status` 字段为目标列对应的状态
2. 更新任务的 `kanban_order` 为目标位置的顺序值
3. 调整目标列中受影响卡片的 `kanban_order`
4. 如果目标状态是 `done`，自动设置 `completed_at = now()`
5. 如果从 `done` 拖出，清除 `completed_at`

**列内排序**：
1. 仅更新受影响卡片的 `kanban_order` 值
2. 使用间隔算法（如每次排序使用中间值），避免频繁重排全部顺序

**乐观更新**：
1. 拖拽释放后立即在 UI 上反映变更（更新 Zustand store）
2. 异步调用 Tauri command 持久化到 SQLite
3. 如果持久化失败，回滚 UI 状态并显示错误 Toast

#### 优先级筛选
- 多选下拉：Urgent / High / Medium / Low

- 全选等于无筛选

#### 标签筛选
- 多选下拉：列出所有标签，带颜色标识

- 支持「无标签」选项

#### 截止日期筛选
- 预设选项：已过期 / 今天 / 明天 / 本周 / 下周 / 无截止日期
- 自定义日期范围（起始日期 - 结束日期）

### 8.4 筛选效果

筛选激活后，不匹配筛选条件的卡片**不会隐藏**，而是变为半透明：

| 状态 | 效果 |
|------|------|
| 匹配筛选 | 正常显示（opacity: 1） |
| 不匹配 | opacity: 0.3，不可拖拽，不可点击 |

这样做的好处：
- 用户仍能看到全局卡片分布
- 列的计数显示匹配数/总数（如 `3/5`）
- 不会因为筛选导致列看起来空荡

**清除筛选**：工具栏筛选器旁显示「清除全部筛选」按钮（仅有活跃筛选时显示）

### 8.5 搜索

**搜索框规格**：
- 宽度：200px（聚焦时展开到 300px，200ms ease）
- placeholder：`搜索任务...`
- 图标：Lucide `Search`（16px）
- 防抖：300ms

**搜索逻辑**：
- 对任务标题进行模糊匹配
- 匹配方式与筛选相同：不匹配的卡片 opacity: 0.3
- 搜索词高亮：匹配的卡片标题中搜索关键词加粗黄色背景标注（`<mark>` 标签）

## 12. 性能优化

### 12.1 虚拟滚动

当单列卡片数超过 50 时，启用垂直虚拟滚动（TanStack Virtual）：
- 只渲染可视区域内的卡片 + 上下各 5 个缓冲卡片
- 列外部容器高度固定，内部使用虚拟滚动
- 滚动时平滑渲染，无闪烁

### 12.2 拖拽性能

- 使用 CSS transform 而非 DOM position 移动元素
- 拖拽过程中不触发 React 重渲染（dnd-kit 内部通过 transform 实现）
- DragOverlay 使用 `position: fixed`，脱离文档流
- 碰撞检测使用 `closestCorners` 算法（性能优于 `rectIntersection`）

### 12.3 批量更新

- 拖拽排序后，批量更新受影响的 `kanban_order`，一次 IPC 调用
- 使用 `requestAnimationFrame` 合并 UI 更新
- Zustand 使用 selector 和 shallow 比较避免不必要的重渲染

## 13. 无障碍（Accessibility）

### 13.1 键盘导航

- 所有卡片和列可通过 Tab 键聚焦
- 聚焦的卡片显示明显的蓝色聚焦环（2px solid #3B82F6，offset 2px）
- 聚焦卡片后：
  - Enter 打开详情
  - Space 开始拖拽（dnd-kit 内置键盘拖拽支持）
  - 方向键在拖拽模式下移动卡片

### 13.2 屏幕阅读器

- 列和卡片使用合适的 ARIA 角色：列容器 `role="listbox"`，卡片 `role="option"`
- 拖拽时 dnd-kit 提供实时 ARIA live announcements：
  - 开始拖拽：「已拾取任务 [标题]，当前在 [列名] 列，位置 [N]/[总数]」
  - 拖拽到新位置：「已移动到 [列名] 列，位置 [N]/[总数]」
  - 释放：「已放置任务 [标题] 到 [列名] 列」
  - 取消：「拖拽已取消」
- 卡片的优先级和状态使用 `aria-label` 描述

### 13.3 颜色对比度

- 所有文字颜色与背景满足 WCAG 2.1 AA 标准（对比度 >= 4.5:1）
- 优先级不仅通过颜色区分，还通过色条位置和文字标签辅助区分
- 筛选后的半透明卡片（opacity: 0.3）不承载交互功能，仅作为上下文提示
