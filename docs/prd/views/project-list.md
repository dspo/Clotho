# 项目列表视图 - 详细设计文档

## 1. 概述

项目列表视图是用户进入 Clotho 应用后的默认视图，展示所有项目的概览信息。用户可以在此视图中浏览、搜索、筛选项目，查看每个项目的任务进度，并通过展开操作直接查看项目下的任务。

**入口**：侧边栏点击"项目"标题，或应用启动时的默认页面。

**路由**：`/projects`（对应 `src/routes/projects/index.tsx`）

---

## 2. 整体布局

### 2.1 页面结构

```
┌──────────────────────────────────────────────────────────────────────┐
│  我的项目                                                            │
│                                                                      │
│  ┌─ 工具栏 ────────────────────────────────────────────────────────┐ │
│  │ [+ 新建项目]   [卡片|列表]   状态: 全部 ▾   排序: 默认 ▾   [🔍] │ │
│  └────────────────────────────────────────────────────────────────┘ │
│                                                                      │
│  ┌─ 内容区 ────────────────────────────────────────────────────────┐ │
│  │                                                                  │ │
│  │  （卡片模式或行模式的项目列表）                                    │ │
│  │                                                                  │ │
│  └────────────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────────────┘
```

### 2.2 区域说明

| 区域 | 说明 |
|------|------|
| 页面标题 | 固定显示"我的项目"，字号 24px，font-semibold |
| 工具栏 | 新建按钮、视图切换、筛选、排序、搜索 |
| 内容区 | 根据视图模式切换卡片布局或行列表布局 |

## 3. 工具栏

### 3.1 工具栏布局

```
┌──────────────────────────────────────────────────────────────────────┐
│ [+ 新建项目]          [卡片][列表]    状态: 全部 ▾    排序 ▾    [🔍] │
│ ← 左对齐               ─────────────────────── 右对齐 ──────────── → │
└──────────────────────────────────────────────────────────────────────┘
```

### 3.2 新建项目按钮

- 位置：工具栏最左侧
- 样式：Primary 按钮，蓝色背景，白色文字，图标 `Plus` + 文字"新建项目"
- 快捷键：无（全局 Cmd+N 创建任务，此处是项目创建）
- 点击后打开「新建项目弹窗」（见第 8 节）

### 3.3 视图模式切换

两个图标按钮组成的 Toggle Group，互斥选择：

| 模式 | 图标 | 说明 |
|------|------|------|
| 卡片模式 | `LayoutGrid` | 网格布局，2-3 列卡片 |
| 行模式 | `List` | 行列表布局，每行一个项目 |

- 默认模式：行模式（信息密度更高）
- 切换时无动画，立即切换
- 选中状态持久化到 AppSettings

### 3.4 状态筛选

下拉菜单（shadcn/ui `Select`），选项：

| 选项 | 值 | 说明 |
|------|------|------|
| 全部 | `all` | 显示进行中 + 已完成的项目（默认） |
| 进行中 | `active` | 仅显示 status=active 的项目 |
| 已完成 | `completed` | 仅显示所有任务已完成的项目 |
| 已归档 | `archived` | 仅显示 status=archived 的项目 |

说明：
- "已完成"是计算状态，指项目内所有任务的 status 均为 done 或 cancelled
- 默认选"全部"时不显示已归档项目，需主动切换到"已归档"查看

### 3.5 排序

下拉菜单，选项：

| 选项 | 排序逻辑 |
|------|---------|
| 默认 | 按 sort_order 字段（手动拖拽排序） |
| 按名称 | 按 name 字母序 A-Z |
| 按名称倒序 | 按 name 字母序 Z-A |
| 按创建时间 | 按 created_at 降序（最新在前） |
| 按截止日期 | 按项目内最近截止任务的日期升序 |
| 按进度 | 按完成百分比降序（进度最高在前） |

说明：
- 选择"默认"排序时，行模式下支持拖拽排序
- 选择其他排序方式时，拖拽排序不可用

### 3.6 搜索框

- 位置：工具栏最右侧
- 默认显示为搜索图标按钮（`Search` 图标）
- 点击后展开为输入框，宽度 240px，带清除按钮
- 搜索匹配：项目名称 + 项目描述
- 防抖：300ms
- 匹配方式：不区分大小写的子字符串匹配
- 快捷键：Cmd+F 聚焦搜索框
- 按 Esc 清空并收起搜索框

### 4.2 项目行详细设计

每个项目行高度 72px，分为两行信息：

```
┌────────────────────────────────────────────────────────────────────┐
│ ▶  ●  项目名称                              8/12 完成  ████░░░░░  │  ← 第一行
│       项目描述摘要文字（单行截断）...         截止: 03-15    进行中  │  ← 第二行
└────────────────────────────────────────────────────────────────────┘
  │  │  │                                      │          │       │
  │  │  │                                      │          │       └─ 状态标签
  │  │  │                                      │          └──────── 进度条
  │  │  │                                      └───────────────── 任务统计
  │  │  └──────────────────────────────────────────────────────── 项目名称
  │  └─────────────────────────────────────────────────────────── 颜色圆点
  └────────────────────────────────────────────────────────────── 展开箭头
```

**各元素规格**：

| 元素 | 规格 |
|------|------|
| 展开箭头 | `ChevronRight` 图标，16x16px，折叠时向右，展开时向下旋转90度，动画 150ms |
| 颜色圆点 | 12x12px 圆形，填充项目颜色（project.color） |
| 项目名称 | 14px，font-medium，单行截断，最大宽度占可用空间 |
| 项目描述 | 13px，text-muted-foreground，单行截断（text-overflow: ellipsis） |
| 任务统计 | 13px，格式 "x/y 完成"，x=已完成任务数，y=总任务数 |
| 进度条 | 宽度 120px，高度 6px，圆角，背景 muted，填充色根据进度渐变 |
| 截止日期 | 13px，text-muted-foreground，显示项目内最近截止日期，格式 MM-DD |
| 状态标签 | Badge 组件，见下方颜色定义 |

**进度条颜色**：
- 0%：灰色（muted）
- 1%-49%：蓝色（#3B82F6）
- 50%-79%：琥珀色（#F59E0B）
- 80%-99%：绿色（#10B981）
- 100%：绿色（#10B981）+ 完成勾选图标

**状态标签颜色**：

| 状态 | 标签文字 | 背景色 | 文字色 |
|------|---------|--------|--------|
| 进行中 | 进行中 | blue-50 / blue-950 | blue-700 / blue-300 |
| 已完成 | 已完成 | green-50 / green-950 | green-700 / green-300 |
| 已归档 | 已归档 | gray-100 / gray-800 | gray-500 / gray-400 |

（斜杠前为亮色主题，斜杠后为暗色主题）

### 4.3 项目行交互

**悬停效果**：
- 行背景变为 accent/5（极浅高亮）
- 右侧显示操作按钮组（编辑、归档、删除），替代截止日期和状态标签位置

```
┌────────────────────────────────────────────────────────────────────┐
│ ▶  ●  项目名称 A                    8/12 完成  ████░░░  [✏][📦][🗑]│  ← 悬停时
│       项目描述摘要...                截止: 03-15                    │
└────────────────────────────────────────────────────────────────────┘
```

**操作按钮**（仅悬停时显示）：

| 按钮 | 图标 | Tooltip | 操作 |
|------|------|---------|------|
| 编辑 | `Pencil` | 编辑项目 | 打开编辑弹窗 |
| 归档 | `Archive` | 归档项目 | 弹出确认后归档 |
| 删除 | `Trash2` | 删除项目 | 弹出确认弹窗 |

**点击行为**：
- 点击项目行（非按钮区域）：进入该项目的默认视图（看板视图）
- 点击展开箭头：展开/折叠项目下的任务列表（不跳转）

**右键菜单**：
- 打开项目
- 重命名
- 设置颜色 -> （颜色选择子菜单）
- 添加到收藏 / 从收藏移除
- 归档项目
- 删除项目

**拖拽排序**（仅"默认"排序时可用）：
- 拖拽手柄：行最左侧区域（展开箭头左边），悬停时显示 `GripVertical` 图标
- 拖拽时行半透明跟随鼠标
- 目标位置显示蓝色插入线（2px，primary 色）
- 释放后更新 sort_order，平滑动画 200ms

## 5. 展开/折叠 - 任务列表

### 5.1 展开状态布局

```
┌────────────────────────────────────────────────────────────────────┐
│ ▼  ●  项目名称 A                    8/12 完成  ████████░░░░  进行中│
│       项目描述摘要...                截止: 03-15                    │
├────────────────────────────────────────────────────────────────────┤
│    ☐  实现用户登录功能            🟡 进行中   🔴 紧急   截止 03-25  │  ← 任务行
│    ☐  优化首页加载速度            🔵 待办     🟡 中等   截止 03-28  │
│    ☑  设计数据库模型              🟢 已完成   🟠 高     截止 03-10  │
│    ☐  编写 API 文档              ⚪ 待规划   🔵 低     —          │
│    + 添加任务...                                                   │  ← 行内创建
├────────────────────────────────────────────────────────────────────┤
│ ▶  ●  项目名称 B                    5/5 完成   ██████████████ 已完成│  ← 下一个项目
└────────────────────────────────────────────────────────────────────┘
```

### 5.2 任务行设计

展开后在项目行下方缩进显示任务列表，每行高度 40px：

```
     ☐  任务标题                      状态标签   优先级标签   截止日期
     │  │                              │          │           │
     │  └─ 14px，单行截断               │          │           └─ 13px, MM-DD 格式
     │                                  │          └─ 优先级颜色点 + 文字
     └─ Checkbox，点击切换完成状态       └─ 状态图标 + 文字
```

**任务行元素规格**：

| 元素 | 规格 |
|------|------|
| 缩进 | 左侧 padding 40px（相对于项目行，表达层级关系） |
| Checkbox | shadcn/ui Checkbox，16x16px |
| 任务标题 | 14px，已完成任务添加删除线 + text-muted-foreground |
| 状态标签 | 状态圆点 8px + 文字 12px，颜色同 PRD 定义 |
| 优先级标签 | 优先级圆点 8px + 文字 12px，颜色同 PRD 定义 |
| 截止日期 | 13px，过期日期显示为 destructive 色（红） |

**Checkbox 行为**：
- 未完成任务（backlog/todo/in_progress）：点击 checkbox 将状态改为 done
- 已完成任务（done）：点击 checkbox 将状态改为 todo
- cancelled 状态任务：checkbox 显示为横线样式（dash），不可点击

**状态图标映射**：

| 状态 | 图标 | 颜色 |
|------|------|------|
| Backlog | `Circle` (空心) | #6B7280 |
| Todo | `CircleDot` | #8B5CF6 |
| In Progress | `Clock` | #F59E0B |
| Done | `CircleCheck` | #10B981 |
| Cancelled | `CircleX` | #EF4444 |

### 5.3 子任务展开（二级展开）

任务行如果有子任务，左侧显示展开箭头：

```
     ▼  ☐  实现用户登录功能            🟡 进行中   🔴 紧急   截止 03-25
           ☐  设计登录页面             🟢 已完成   🟡 中等   截止 03-20
           ☐  接入 OAuth              🔵 待办     🟠 高     截止 03-23
           ☐  编写登录测试             ⚪ 待规划   🔵 低     —
     ▶  ☐  优化首页加载速度            🔵 待办     🟡 中等   截止 03-28
```

- 子任务缩进额外 24px（总缩进 64px）
- 子任务行高度同 40px
- 子任务不再有下一级展开（仅支持一级子任务）

### 5.4 行内快速创建任务

展开状态下，任务列表底部显示创建入口：

```
     + 添加任务...
```

- 点击后转为输入框，获得焦点
- 输入标题后按 Enter 创建任务（默认状态 todo，默认优先级 low）
- 按 Esc 取消创建
- 创建成功后输入框保留，可连续创建
- 创建的任务追加到列表末尾

### 5.5 任务行交互

**点击任务行**：
- 点击任务标题或行空白区域：打开右侧任务详情面板（见第 7 节）
- 点击 checkbox：切换完成状态
- 点击状态标签：弹出状态选择下拉菜单，可直接切换

**悬停效果**：
- 行背景变为 accent/5
- 右侧显示操作图标：`MoreHorizontal`（更多操作）

**右键菜单**（任务行）：
- 打开详情
- 编辑标题
- 设置状态 -> （子菜单：5 种状态）
- 设置优先级 -> （子菜单：4 种优先级）
- 添加标签 ->
- 设置截止日期
- 复制任务
- 删除任务

### 6.2 卡片规格

- 布局：CSS Grid，`grid-template-columns: repeat(auto-fill, minmax(280px, 1fr))`
- 间距：gap 16px
- 卡片最小宽度：280px
- 卡片最大宽度：自适应填充
- 卡片高度：自适应内容，最小 160px
- 卡片圆角：8px（rounded-lg）
- 卡片边框：1px border，默认透明，悬停时 border-primary/20
- 卡片阴影：默认 shadow-sm，悬停时 shadow-md

### 6.3 卡片内容区域

```
┌─────────────────────────────┐
│ ●  项目名称                  │ ← 头部：颜色圆点 + 名称
│                              │
│ 项目描述摘要文字，最多         │ ← 描述：最多2行，超出截断
│ 显示两行截断...              │
│                              │
│ ████████████░░░░  8/12 完成  │ ← 进度条 + 统计
│ 截止: 2026-03-15    [进行中]  │ ← 截止日期 + 状态标签
└─────────────────────────────┘
```

| 区域 | 规格 |
|------|------|
| 头部 | 颜色圆点 12px + 项目名称 15px font-medium，单行截断 |
| 描述 | 13px，text-muted-foreground，最多 2 行（line-clamp-2） |
| 进度条 | 宽度 100%，高度 6px，圆角 |
| 任务统计 | 13px，进度条右侧对齐 |
| 截止日期 | 13px，text-muted-foreground |
| 状态标签 | 同行模式的 Badge 组件 |

### 6.4 卡片交互

**悬停**：
- 卡片边框高亮 border-primary/20
- 阴影加深 shadow-md
- 右上角显示更多操作按钮（`MoreHorizontal`）

**点击**：
- 点击卡片进入该项目的默认视图（看板视图）

**右键菜单**：同行模式的项目右键菜单

**拖拽排序**（仅"默认"排序时可用）：
- 长按 200ms 或拖拽手柄触发拖拽
- 拖拽时卡片半透明跟随
- 目标位置显示蓝色边框高亮
- 释放后卡片平滑移动到新位置

**注意**：卡片模式不支持展开查看任务（信息密度限制），需切换到行模式使用展开功能。

## 7. 任务详情面板

### 7.1 面板布局

点击任务行后从右侧滑出的详情面板：

```
                                        ┌─ 详情面板 ─────────────────┐
                                        │ ✕                          │
                                        │                            │
                                        │ 任务标题（可编辑）           │
                                        │ ─────────────────────────  │
                                        │                            │
                                        │ 状态      [🟡 进行中 ▾]    │
                                        │ 优先级    [🔴 紧急 ▾]      │
                                        │ 标签      [#后端] [+ 添加]  │
                                        │ 截止日期  [2026-03-25 📅]   │
                                        │ 所属项目  项目名称 A        │
```

## 12. 响应式与性能

### 12.1 响应式设计

- 卡片模式响应列数：
  - 窗口宽度 >= 1200px（去除侧边栏后）：3 列
  - 窗口宽度 >= 800px：2 列
  - 窗口宽度 < 800px：1 列
- 行模式自适应宽度，各元素使用 flex 布局

### 12.2 性能策略

- 项目数量通常不大（<100），无需虚拟滚动
- 展开的任务列表如超过 50 条，使用 TanStack Virtual 虚拟滚动
- 展开/折叠使用 CSS transition（height auto 动画可用 `grid-template-rows: 0fr -> 1fr` 技巧）
- 搜索防抖 300ms
- 进度计算在 Rust 侧完成，前端直接使用计算结果
